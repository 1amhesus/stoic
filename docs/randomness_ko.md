/* Shaking 전략 vs. 비트코인 엔트로피 사용 – 비교 요약 */

/*
1. 비트 시프트 + XOR (비트 이동 + XOR 혼합)

   → 전략: 타임스탬프(예: `now`)의 서로 다른 비트 구간을 오른쪽 시프트와 XOR로 섞어 하위 비트만 변화할 때 동일한 출력이 반복되는 문제를 방지한다.

   - 비트코인과의 유사성:

     · Bitcoin Core RNG 설계 논의(#14623)에서는 “fast seed” 단계에서 스택 포인터, 고해상도 타임스탬프와 같은 빠르게 변화하는 엔트로피를
       
       혼합하는 것을 제안한다. 이러한 값들은 시프트/XOR 연산을 통해 처리되는 경우가 많다.
           
       출처: https://github.com/bitcoin/bitcoin/issues/14623

        -> 사실 확인 필요: [x]

           "fast" seeding은 GetRandBytes 호출시 실행되며 스택 포인터 + 고정밀 타임스탬프를 혼합함 (#14955 병합, src/random.h)
*/

/*
2. LCG 스타일 수식 (선형 합동 생성기)

   → 전략: 숫자 시드(예: `now`)에 상수 곱셈과 덧셈을 적용하고 모듈로 연산을 통해 일정한 변동성을 생성한다.

   - 비트코인과의 관련성:

     · 비트코인 코어에서는 난수 생성에 LCG 방식이 명시적으로 사용되지 않고 대신 SHA512와 같은 해시 기반 혼합이 고엔트로피 상황에서 선호된다.

     · RNG 설계에서는 OS 엔트로피 소스를 혼합한 뒤 암호학적 해시를 적용하는 방식이 언급된다. 이는 LCG보다 훨씬 높은 예측 불가능성을 제공한다.
*/

/*
3. Rotate + XOR (비트 회전 + XOR)

   → 전략: 단순 시프트 대신 비트를 좌/우로 회전한 뒤 XOR을 적용하여 더 다양한 결과를 생성한다.

   - 비트코인과의 관련성:

     · 코드에서 명시적으로 rotate 연산이 드러나지는 않지만 엔트로피 수집(#14623) 및 RNG 설계 논의에서 저수준 비트 조작이 사용됨을 암시한다.

        -> 사실 확인 필요: [ ]

     · 특히 시간, 스택, 프로세스 기반 엔트로피를 혼합할 때
       rotate + XOR 패턴은 개념적으로 일치한다.
*/

/*
4. 시스템 소스 결합 (시간, PID, OS 정보, 메모리 등)

   → 전략: 단일 값이 아니라 여러 엔트로피 소스를 혼합한다.(타임스탬프, 프로세스 ID, 메모리 정보 등)

   - 비트코인과의 유사성:

     · Bitcoin Core HD 지갑 시드 생성은 OS 엔트로피(getrandom/urandom), CPU 명령어(RDRAND/RDSEED), 환경 잡음(힙/스택 주소, sleep 시간 등)을 결합하여 사용한다.

        -> 사실 확인 필요: [ ]

     · RNG 설계(#14623)에는 “fast”, “medium”, “slow” 시딩 레이어를 정의하고 각 레이어가 시스템 성능 카운터, 애플리케이션 이벤트 등 다양한 엔트로피를 추가한다.

        -> 사실 확인 필요: [ ]
*/

/*
== 요약 분석 =================================================================================================================

- 비트코인은 LCG와 같은 단순 수치 변환보다는 강력한 엔트로피 소스와 암호학적 혼합(SHA512 등)을 선호한다.

- CLI 도구(`stoic-cli`) 관점에서는 전략 1 (비트 시프트 + XOR) 또는 전략 4 (시스템 소스 결합)이 의존성을 최소화하면서도 충분한 “체감 랜덤성”을 제공한다.

- 만약 암호학적 수준의 예측 불가능성이나 공격자 모델을 고려해야 한다면 비트코인 방식처럼,

    → 다중 엔트로피 소스 사용

    → 안전한 해시 또는 OS RNG 활용

  이 필요하다.

=========================================================================================================================
*/
